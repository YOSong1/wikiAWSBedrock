앞선 챕터에서 우리는 Vertex AI를 사용하기 위한 튼튼한 개발 환경을 구축했습니다. 이제 본격적으로 구글의 최신 생성형 AI 모델인 **Gemini(제미나이)**를 우리 코드 안으로 불러올 차례입니다.

하지만 코드를 작성하기 전에 먼저 고민해야 할 것이 있습니다. **"어떤 Gemini 모델을 써야 할까?"** 그리고 **"어떻게 말해야 AI가 내 의도대로 똑똑하게 대답할까?"**라는 점입니다. 이번 챕터에서는 Gemini 모델 패밀리의 특징을 이해하고, Vertex AI SDK를 사용하여 텍스트 생성부터 문맥을 기억하는 챗봇, 그리고 AI에게 페르소나를 부여하는 **시스템 지침(System Instruction)**까지 단계별로 구현해 보겠습니다.

---

## 5-1 Gemini 모델 패밀리(Flash/Pro/Ultra) 이해

Gemini는 단일 모델이 아니라, 각기 다른 크기와 성능, 비용 효율성을 가진 **'모델 패밀리(Model Family)'**로 구성되어 있습니다. 개발자는 해결하려는 문제의 난이도와 예산에 맞춰 가장 적합한 모델을 선택해야 합니다. 현재 Vertex AI에서 제공하는 주요 모델 라인업을 살펴보겠습니다.

---

---

### 5-1-1 Gemini Flash: 속도와 효율성의 챔피언

**Gemini Flash**는 이름처럼 가장 빠르고 가벼운 모델입니다. 방대한 양의 데이터를 빠르게 처리해야 하거나, **응답 속도(Latency)**가 중요한 실시간 애플리케이션에 적합합니다. 비용 또한 가장 저렴하기 때문에, 단순한 텍스트 요약이나 데이터 추출, 대규모 반복 작업에 최적화되어 있습니다. **"가성비"**가 중요한 프로젝트라면 Flash가 정답입니다.

---

### 5-1-2 Gemini Pro: 균형 잡힌 올라운더

**Gemini Pro**는 성능과 비용 사이에서 가장 이상적인 균형을 맞춘 모델입니다. 복잡한 추론, 창의적인 글쓰기, 코드 생성 등 대부분의 생성형 AI 작업에서 우수한 성능을 보여줍니다. Vertex AI를 사용하는 대다수의 엔터프라이즈 애플리케이션이 이 **Pro 모델**을 표준으로 사용합니다. 특별한 이유가 없다면 개발을 시작할 때 Pro 모델을 선택하는 것이 가장 무난합니다.

---

### 5-1-3 Gemini Ultra: 최고의 성능과 심층 추론

**Gemini Ultra**는 가장 거대하고 강력한 모델입니다. 고도로 복잡한 수학 문제 해결, 깊은 문맥 이해, 전문적인 코딩 작업 등 최고의 지능이 필요할 때 사용됩니다. 다만, Flash나 Pro에 비해 비용이 높고 응답 속도가 다소 느릴 수 있으므로, 난이도가 매우 높은 특수 목적의 태스크에 선별적으로 사용하는 것이 좋습니다.

> 📷 **[이미지]** Gemini 모델 패밀리(Flash, Pro, Ultra)를 비교하는 도표나 그래프 이미지를 넣어주세요. X축은 성능(복잡성), Y축은 비용/속도 등을 나타내어 각 모델의 포지셔닝을 한눈에 보여주는 그림이 좋습니다.

## 5-2 Vertex AI SDK로 텍스트 생성/요약/변환 호출하기

모델을 선택했다면, 이제 Python 코드로 직접 Gemini에게 일을 시켜보겠습니다. 가장 기본적인 기능은 **'텍스트 생성(Generate Content)'**입니다. 이는 **질문(Prompt)**을 던지면 답변을 받는 단발성 작업입니다.

---

---

### 5-2-1 기본 텍스트 생성 코드 구조

이전 챕터에서 설정한 환경에서 다음 코드를 작성해 봅니다. 여기서 핵심은 **GenerativeModel 객체**를 생성할 때 우리가 사용할 모델의 이름(예: gemini-1.5-flash-001)을 정확히 지정하는 것입니다.



```python
import vertexai
from vertexai.generative_models import GenerativeModel

# 1. Vertex AI 초기화 (프로젝트 ID와 리전 설정)
vertexai.init(project="your-project-id", location="us-central1")

# 2. 모델 로드 (가성비 좋은 Flash 모델 선택)
model = GenerativeModel("gemini-1.5-flash-001")

# 3. 프롬프트 작성 및 요청
prompt = "인공지능 개발자가 되기 위해 공부해야 할 3가지를 요약해줘."
response = model.generate_content(prompt)

# 4. 결과 출력
print(response.text)
```

---

---

### 5-2-2 생성 제어 파라미터 (Configuration)

단순히 질문만 던지는 것을 넘어, 답변의 스타일을 제어할 수도 있습니다. **generation_config** 매개변수를 사용하면 모델의 **'창의성'**이나 **'답변 길이'**를 조절할 수 있습니다.

**temperature (온도)**

0에서 1 사이의 값입니다. 낮을수록(0.2) 사실적이고 결정적인 답변을, 높을수록(0.8) 창의적이고 다양한 답변을 내놓습니다.

**max_output_tokens**

모델이 생성할 답변의 최대 길이를 제한합니다.



```python
from vertexai.generative_models import GenerationConfig

config = GenerationConfig(
    temperature=0.7,  # 적당한 창의성 부여
    max_output_tokens=500  # 답변 길이 제한
)

response = model.generate_content(
    "창의적인 마케팅 슬로건 5개 만들어줘",
    generation_config=config
)
```

 > 📷 **[이미지]** VS Code에서 위 코드를 실행하고, 터미널 창에 Gemini가 생성한 답변(3가지 요약 등)이 깔끔하게 출력된 스크린샷을 넣어주세요.

## 5-3 ChatSession 패턴으로 멀티턴 대화 구현

앞서 배운 **generate_content**는 기억력이 없습니다. "안녕?"이라고 한 뒤, 다시 "방금 내가 뭐라고 했지?"라고 물으면 대답하지 못합니다. 이를 **'싱글턴(Single-turn)'** 방식이라고 합니다. 하지만 챗봇처럼 이전 대화를 기억하며 꼬리에 꼬리를 무는 대화를 하려면 **'멀티턴(Multi-turn)'** 방식이 필요합니다.

---

---

### 5-3-1 ChatSession 객체의 이해

Vertex AI SDK는 대화의 **맥락(Context)**을 자동으로 관리해 주는 **ChatSession 객체**를 제공합니다. 개발자가 일일이 이전 대화 내역을 리스트에 저장해서 다시 보낼 필요 없이, 단순히 채팅 세션을 시작(start_chat)하기만 하면 SDK가 알아서 기억을 관리해 줍니다.

---

---

### 5-3-2 멀티턴 챗봇 코드 구현

다음은 사용자가 '종료'라고 입력할 때까지 계속 대화를 이어가는 간단한 **챗봇 코드**입니다.



```python
# 1. 채팅 세션 시작 (history를 자동으로 관리함)
chat = model.start_chat()
print("Gemini 챗봇과 대화를 시작합니다. (종료하려면 'quit' 입력)")

while True:
    # 사용자 입력 받기
    user_input = input("User: ")
    if user_input.lower() == "quit":
        break

    # 2. 채팅 메시지 전송 (send_message 사용)
    response = chat.send_message(user_input)

    # 3. 모델 응답 출력
    print(f"Gemini: {response.text}")
    print("-" * 20)
```

이 코드를 실행하고 "내 이름은 김철수야"라고 말한 뒤, "내 이름이 뭐지?"라고 물어보세요. 놀랍게도 Gemini는 앞선 대화를 기억하고 "**김철수입니다**"라고 대답할 것입니다. **chat 객체** 내부의 **history 속성**에 대화 내용이 차곡차곡 쌓이기 때문입니다.

> 📷 **[이미지]** 터미널에서 실행된 멀티턴 대화의 예시 화면을 캡처해 주세요. (User: 안녕, 나는 학생이야 -> Gemini: 반가워요 -> User: 내 직업이 뭐지? -> Gemini: 당신은 학생입니다)와 같이 문맥을 기억하는 모습이 담겨야 합니다.

## 5-4 System Instruction과 기본 프롬프트 설계 패턴

AI를 단순히 **'똑똑한 검색 엔진'**으로 쓰는 것을 넘어, 특정한 **'역할(Persona)'**이나 **'규칙'**을 부여하고 싶을 때가 있습니다. 예를 들어 "너는 친절한 영어 선생님이야"라거나 "모든 답변은 JSON 형식으로만 해"라고 지시하는 것입니다. 이를 위해 사용하는 것이 바로 **시스템 지침(System Instruction)**입니다.

---

---

### 5-4-1 System Instruction의 개념

**System Instruction**은 사용자가 대화를 시작하기 전에, 모델에게 미리 주입하는 **'절대적인 행동 강령'**과 같습니다. 이는 일반적인 프롬프트보다 **우선순위가 높게 처리**되므로, 모델이 대화 내내 그 역할을 유지하도록 강제하는 데 매우 효과적입니다.

---

---

### 5-4-2 시스템 지침 적용하기

모델을 로드하는 단계에서 **system_instruction** 파라미터를 추가하여 설정합니다.



```python
# 시스템 지침 정의: 수학 선생님 페르소나 부여
system_prompt = """
당신은 초등학생에게 수학을 가르치는 친절한 선생님입니다.
다음 규칙을 반드시 지키세요:
1. 어려운 수학 용어 대신 쉬운 비유를 사용하세요.
2. 정답을 바로 알려주지 말고, 힌트를 주어 스스로 풀게 유도하세요.
3. 항상 존댓말을 사용하고 칭찬을 많이 해주세요.
"""

# 모델 로드 시 시스템 지침 주입
model = GenerativeModel(
    "gemini-1.5-pro-001",
    system_instruction=[system_prompt]
)

# 테스트
response = model.generate_content("선생님, 피타고라스 정리가 뭐예요?")
print(response.text)
```

이 코드를 실행하면, Gemini는 딱딱한 사전적 정의를 읊는 대신, **"삼각형 친구들이 모여서..."**와 같은 식으로 초등학생 눈높이에 맞춰 친절하게 설명해 줄 것입니다. 이처럼 **System Instruction**은 AI 애플리케이션의 성격을 결정짓는 가장 중요한 요소 중 하나입니다.

> 📷 **[이미지]** 동일한 질문("피타고라스 정리가 뭐예요?")에 대해, 시스템 지침이 없을 때(일반적인 답변)와 시스템 지침(초등 교사 페르소나)이 적용되었을 때의 답변 차이를 비교하는 이미지를 넣어주세요.
